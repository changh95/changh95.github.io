  • Input лҚ°мқҙн„°м—җлҠ” л‘җк°Җм§Җ мў…лҘҳмқҳ лҚ°мқҙн„°к°Җ м„һм—¬мһҲмқҢ.

    • Inlier лҚ°мқҙн„° - кё°н•ҳн•ҷм ҒмңјлЎңлӮҳ нҶөкі„м ҒмңјлЎң мҳ¬л°”лҘё лҚ°мқҙн„°.
    • лӢЁмҲңнһҲ лҚ°мқҙн„°л§Ң лҙҗм„ңлҠ” inlier / outlier кө¬л¶„н•ҳкё° м–ҙл ӨмӣҖ.
  • Model parameterлҘј м •нҷ•н•ҳкІҢ кі„мӮ°н•ҳкё° мң„н•ҙм„ңлҠ” Inlier лҚ°мқҙн„°л§Ң мӮ¬мҡ©н•ҙм•јн•Ё.

  • Model parameterлҘј кі„мӮ°н•ҳкё° мң„н•ҙ, м „мІҙ лҚ°мқҙн„° мӨ‘ inlierмҷҖ outlierлҘј кө¬л¶„н•ҳм—¬ inlier лҚ°мқҙн„°л§Ң кі„мӮ°м—җ мӮ¬мҡ©н•ҙм•јн•Ё.

    • кі„мӮ°мқ„ н•ҙлҙӨлҚ”лӢҲ 10к°ңмқҳ лҚ°мқҙн„°к°Җ error tresholdлҘј л„ҳм§Җ м•Ҡм•ҳлӢӨкі  н•ҙліҙмһҗ. мҡ°лҰ¬лҠ” мқҙ лҚ°мқҙн„°мқҳ к°ҜмҲҳлҘј вҖҳinlierмқҳ мҲҳвҖҷлЎң м·Ёкёүн•ҳл©°, мқҙ к°’мқ„ мҡ°лҰ¬лҠ” вҖҳbest model scoreвҖҷлЎң м ҖмһҘн•ңлӢӨ.
  • л‘җлІҲм§ё лЈЁн”„лҘј мӢңмһ‘н•ңлӢӨ. лӢӨмӢң 100к°ң лҚ°мқҙн„° мӨ‘ 4к°ңлҘј л¬ҙмһ‘мң„лЎң лҪ‘лҠ”лӢӨ.
    • мқҙ 4к°ңмқҳ лҚ°мқҙн„°лҘј кё°л°ҳмңјлЎң Homography matrixлҘј кі„мӮ°н•ңлӢӨ.
    • лӢӨмӢң н•ҙлӢ№ homography matrixлҘј лӘЁл“  input лҚ°мқҙн„°м—җ м Ғмҡ©н•ҳкі  inlierмқҳ мҲҳлҘј мёЎм •н•ңлӢӨ.

  • м„ёлІҲм§ё лЈЁн”„лҘј мӢңмһ‘н•ңлӢӨ. лӢӨмӢң 100к°ң лҚ°мқҙн„° мӨ‘ 4к°ңлҘј л¬ҙмһ‘мң„лЎң лҪ‘лҠ”лӢӨ.
    • мқҙ 4к°ңмқҳ лҚ°мқҙн„°лҘј кё°л°ҳмңјлЎң Homography matrixлҘј кі„мӮ°н•ңлӢӨ.
    • лӢӨмӢң н•ҙлӢ№ homography matrixлҘј лӘЁл“  input лҚ°мқҙн„°м—җ м Ғмҡ©н•ҳкі  inlierмқҳ мҲҳлҘј мёЎм •н•ңлӢӨ.
    • мқҙлІҲм—җлҠ” 25к°ңмқҳ inlier лҚ°мқҙн„°к°Җ мёЎм •лҗҳм—Ҳкі , м§ҖлӮң лЈЁн”„мқҳ 50к°ңліҙлӢӨ м ҒлӢӨ. мҰү, мқҙ лӘЁлҚёмқҖ м§ҖлӮң лЈЁн”„ліҙлӢӨ лҚ” л¶Җм •нҷ•н•ң лӘЁлҚёмқҙлӢҲ, вҖҳbest model scoreвҖҷлҠ” ліҖн•ҳм§Җ м•ҠлҠ”лӢӨ.
  • мқҙ кіјм •мқ„ л°ҳліөн•ңлӢӨ.

RANSAC кіөмӢқ

  • T: мғҳн”Ңл§Ғмқҳ нҡҹмҲҳ
  • p: мҡ°лҰ¬к°Җ кі лҘё лҚ°мқҙн„°к°Җ inlierмқё нҷ•лҘ 
  • e: м „мІҙ лҚ°мқҙн„°мқҳ inlier:outlier л№„мңЁ
  • s: Minimal setмқ„ л§Ңл“Өкё° мң„н•ң лҚ°мқҙн„°мқҳ к°ҜмҲҳ
  • p, e, sлҠ” ліҙнҶө мҡ°лҰ¬к°Җ кі лҰ„.
    • мҳҲмӢңвҖҰ
    • p = 0.99 (99%мқҳ нҷ•лҘ лЎң м •нҷ•н•ң лӘЁлҚёмқ„ м–»кі мӢ¶лӢӨ)
    • e = 0.50 (м „мІҙ лҚ°мқҙн„° мӨ‘ inlier:outlier л№„мңЁмқҙ 50:50мқҙлӢӨ. [л¬јлЎ  м ҲлҢҖлЎң мқҙ к°’мқҖ м •нҷ•н•ҳкІҢ м•Ң мҲҳ м—ҶлӢӨ.])
    • s = 3,4,5,7,8вҖҰ (p3p? Homography? Nister E? 7-point F? 8-point F?)

    • Threshold к°’мқ„ мң м Җк°Җ м§Ғм ‘ нҠңлӢқн•ҙм•јн•Ё (i.e. мң м Җмқҳ кІҪн—ҳм—җ мқҳмЎҙн•Ё)

M-EstimatorлһҖ?

  • м»ҙн“Ён„° л№„м „м—җм„ңлҠ” мў…мў… Least-squares optimisationмқ„ нҶөн•ҙ Maximum-likelihood Estimation (MLE)лҘј мҲҳн–үн•ҳм—¬ model parameterлҘј м¶”м •н•ҳкё°лҸ„ н•ңлӢӨ.
    • Least-sqauresлҠ” лӘЁл“  лҚ°мқҙн„° нҸ¬мқёнҠёк°Җ unbiasн•ң л…ёмқҙмҰҲлҘј к°Җм§„лӢӨлҠ” кІғмқ„ м „м ңлЎң мӮјлҠ”лӢӨ.
    • н•ҳм§Җл§Ң biasк°Җ нҸ¬н•Ёлҗң лҚ°мқҙн„° нҸ¬мқёнҠёк°Җ кі„мӮ°м—җ нҸ¬н•Ёлҗҳл©ҙ least-squares кіјм •мқҙ л¬ҙл„Ҳм§ҖкІҢ лҗңлӢӨ.
  • мқҙлҹ¬н•ң biasлҘј к°Җм§„ лҚ°мқҙн„° нҸ¬мқёнҠёлҘј м°ҫм•„лӮҙкі  кі„мӮ°м—җм„ң м ңмҷён•ҳлҠ” кё°мҲ л“Өмқ„ robust estimationмқҙлқјкі  н•ңлӢӨ.
    • мқҙ мӨ‘, kernel кё°лІ•мқ„ мӮ¬мҡ©н•ҳм—¬ лҚ°мқҙн„° нҸ¬мқёнҠёл“Өмқҳ residual к°’м—җ weightлҘј кұём–ҙ optimisationмқ„ мҲҳн–үн•  мҲҳ мһҲкІҢ н•ҙмЈјлҠ” кё°мҲ мқҙ M-estimatorмқҙлӢӨ.

  • кё°мЎҙмқҳ л°©мӢқмқҖ м–ҙл–Ө model parameter мқ„ м¶”м •н•ҳкі , н•ҙлӢ№ лӘЁлҚёмқҳ м •нҷ•лҸ„лҠ” лӘЁл“  лҚ°мқҙн„° м—җ лҢҖн•ҙ residual лҘј кө¬н•ҙ нҸүк°Җн•ҳмҳҖлӢӨ.
    • мҲҳмӢқмңјлЎң н‘ңнҳ„н•ҳл©ҙ лӢӨмқҢкіј к°ҷлӢӨ:
    • (н•ҳм§Җл§Ң мқҙ к°’мқҖ biasлҘј к°Җм§„ лҚ°мқҙн„°м—җм„ңлҠ” 1. нҒ° к°’мқ„ к°Җм§Җл©°, 2. нҢЁн„ҙмқ„ к°Җм§Җл©° лӮҳнғҖлӮ¬кі , мқҙлҠ” optimisationмқ„ м–ҙл өкІҢ н•ҳмҳҖлӢӨ).
  • M-estimatorлҠ” мӮ¬м „м—җ м •н•ҙл‘” м»Өл„җмқҳ лӘЁм–‘мқ„ л”°лқј, residual к°’м—җ л”°лҘё weightлҘј мғҲлЎң м§Җм •н•ңлӢӨ.
    • мҲҳмӢқмңјлЎң н‘ңнҳ„н•ҳл©ҙ лӢӨмқҢкіј к°ҷлӢӨ.
  • м»Өл„җмқҳ лӘЁм–‘мқҖ мқҙ л§ҒнҒ¬, к·ёлҰ¬кі  м•„лһҳ к·ёлҰјм—җлҸ„ н‘ңнҳ„лҗҳм–ҙмһҲлӢӨ.
    • м•„л¬ҙлҹ° kernelлҸ„ мӮ¬мҡ©н•ҳм§Җ м•ҠмқҖ pure least-squaresмқҳ кІҪмҡ°, residual к°’мқҖ нҳ•нғңлЎң кі„мӮ°лҗңлӢӨ.
      • мқҙлҠ” м „мІҙлӢӨмҲҳмқҳ residualмқҙ м–ҙл– н•ң optimal к°’мқ„ н–Ҙн•ҳкі  мһҲм–ҙлҸ„, м–ҙл–Ө лҶ’мқҖ к°’мқ„ к°Җм§„ outlier лҚ°мқҙн„°к°Җ лӮҳнғҖлӮңлӢӨл©ҙ м „мІҙм Ғмқё optimisation л°©н–ҘмқҖ мқҙ outlierмқҳ к°’мқ„ мӨ„мқҙкё° мң„н•ң л°©н–ҘмңјлЎң нқҳлҹ¬к°„лӢӨлҠ” кІғмқҙлӢӨ (мқҙкІғмқҖ мўӢм§Җ м•ҠлӢӨ).
    • Hard-redescenderлЎң мһҳ м•Ңл Өм§„ TukeyлӮҳ Gemen-McClure kernelмқҳ кІҪмҡ°, нҠ№м • к°’ мқҙмғҒл¶Җн„°лҠ” constantлЎң л°”кҝ”лІ„лҰ°лӢӨ.
      • мқҙлҠ”, нҠ№м • к°’ мқҙмғҒл¶Җн„°лҠ” лӘЁл“  к°’л“Өмқҙ outlierлЎң н‘ңнҳ„лҗҳл©°, лӢЁмқј к°’мқҙ лҶ’лӢӨкі  н•ҙлҸ„ м „мІҙм Ғмқё optimisation л°©н–Ҙм—җ нҒ¬кІҢ мҳҒн–Ҙмқ„ лҒјм№ҳм§Җ лӘ»н•ңлӢӨ. м „мІҙм Ғмқё optimisationмқҳ л°©н–Ҙмқҙ л°”лҖҢкё° мң„н•ҙм„ңлҠ” м „мІҙлӢӨмҲҳмқҳ residualмқҙ нҠ№м •н•ң л°©н–Ҙм—җ optimal к°’мқҙ мһҲлӢӨкі  лҸҷмқҳн•  л•Ңл§Ң к°ҖлҠҘн•ҳлӢӨ.
