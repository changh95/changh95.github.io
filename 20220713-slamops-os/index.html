SLAMOps로의 길 (0) - 운영체제(OS)는 어떤걸 써야할까?

Posted on 2022-08-08 Edited on 2022-08-12 In 1. Spatial AI , 1.1 SLAM Views:

어떤 OS를 기반으로 SLAM을 개발하는게 좋을까?

 

운영체제(OS)란?

Windows, MacOS, Linux, Android, iOS 와 같은 운영체제들이 있다.

 

Wikipedia에 의하면, 운영체제는 사용자의 하드웨어, 시스템 리소스를 제어하고 프로그램에 대한 일반적 서비스를 지원하는 시스템 소프트웨어입니다.

흔히 사용하는 컴퓨터에서는 Windows, MacOS, Linux와 같은 운영체제들을 볼 수 있습니다. (물론 Linux는 커널이지만, 쉬운 분류를 위해 Linux 패밀리를 칭했습니다)

그리고 모바일에서는 Android와 iOS를 쉽게 볼 수 있습니다.

 

SLAM 알고리즘 자체는 운영체제를 가리지 않고 잘 동작하지만, 효율적이고 빠르게 SLAM 시스템을 동작시키기 위해서는 적절한 운영체제를 고를 필요가 있습니다.

이번 글에서는 각각의 OS에서 SLAM을 돌리는 케이스에 대해 이야기해보고, 그래서 어떤 OS를 기반으로 SLAM을 개발하는게 좋은지에 대해 분석해보겠습니다.

 

Windows

격하게 비추 - 99%의 상황에서 Windows로 SLAM을 개발할 일은 없습니다.
딱 한가지 예외로, MS HoloLens 앱을 만들 때 SLAM 알고리즘도 같이 개발하는 경우가 있습니다.

 

단언컨데, 제대로 SLAM을 개발하는 곳에서는 Windows로 개발을 하지 않을겁니다.

혹시나 이 글을 읽고있는 독자님께서도 혹시 Windows 환경에서 Visual Studio를 통해 개발을 하신다거나 Docker/WSL 기반으로 Linux 환경을 만들어서 개발을 하신다면, 꼭 Windows에서 벗어나시기를 강력하게 추천드리는 바 입니다.

SLAM 개발에 Windows가 선호되지 않는 이유를 하나씩 설명해보도록 하겠습니다.

 

첫째, 배포 환경이 Windows가 아닌 경우가 많음

SLAM은 모바일 로보틱스 학문에서 나온 알고리즘이며, 주로 1. 로보틱스, 2. 자율주행차, 3. 드론, 4. VR/AR과 같은 분야에서 사용됩니다.

SLAM이 사용되는 모바일 플랫폼들은 주로 ARM CPU 아키텍처를 사용하는데, 이 아키텍처는 비교적 최근까지도 Windows가 동작하지 않았습니다.

그러다보니 주로 로봇, 자율주행차, 드론, VR/AR 헤드셋에는 Linux 기반 OS가 탑재가 되었고, 종종 VR/AR에서 Android OS를 탑재한 스마트 글래스나 스마트폰 앱으로써 Android / iOS에 탑재된 모습을 볼 수 있습니다.

Windows에서 개발을 하게 되면 결국에는 다른 OS에서 작동하는 버전을 또 다시 만들어야합니다.

 

우리가 Windows에서 개발을 해본다고 합시다.

가장 많이 쓰는 IDE인 Visual Studio를 쓸 수도 있고, 아니면 VS Code 같은 에디터에 여러가지 C++ 플러그인을 다운받아 IDE 처럼 만들 수도 있을겁니다.
